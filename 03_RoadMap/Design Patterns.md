| Category   | Pattern          | Core Idea                              | When to Use                          | Done | Confidence |
| ---------- | ---------------- | -------------------------------------- | ------------------------------------ | ---- | ---------- |
| Behavioral | Strategy         | Swap algorithms at runtime             | Different behaviors (filters, rules) |      |            |
| Behavioral | Observer         | Notify subscribers on change           | Event systems, pub-sub               |      |            |
| Behavioral | Command          | Encapsulate actions as objects         | Undo/redo, task queues               |      |            |
| Behavioral | State            | Behavior changes based on state        | FSM, workflows                       |      |            |
| Behavioral | Visitor          | Add operations without modifying types | ASTs, compilers                      |      |            |
| Structural | Adapter          | Convert one interface to another       | Legacy integration                   |      |            |
| Structural | Decorator        | Add behavior dynamically               | Logging, validation layers           |      |            |
| Structural | Facade           | Simplify complex subsystem             | Clean APIs                           |      |            |
| Structural | Proxy            | Control access / add indirection       | Lazy loading, caching                |      |            |
| Creational | Factory Method   | Create objects via interface           | Decouple creation logic              |      |            |
| Creational | Abstract Factory | Families of related objects            | Cross-platform systems               |      |            |
| Creational | Builder          | Step-by-step object construction       | Complex objects                      |      |            |
| Creational | Singleton        | Single instance                        | Config, logging (use carefully)      |      |            |
